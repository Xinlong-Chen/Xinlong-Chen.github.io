MySQL - 日志系统

redo log（重做日志）

  • redo log 用于保证 crash-safe 能力。

  • 地点：InnoDB 引擎层。

  • WAL 技术（Write-Ahead Logging），关键点就是先写日志，再写磁盘。

binlog（归档日志）

  • MySQL server层实现的一个逻辑日志。

redo log和binlog的异同

这两种日志有以下三点不同。

  • 地点：redo log 是 InnoDB 引擎特有的；binlog 是 MySQL 的 Server 层实现的，所有引擎都可以使用。
  • 类型：redo log 是物理日志，记录的是“在某个数据页上做了什么修改”；binlog 是逻辑日志，记录的是这个语句的原始逻辑，比如“给 ID=2 这一行的 c 字段加 1 ”。
  • 写入方式：redo log 是循环写的，空间固定会用完；binlog 是可以追加写入的。“追加写”是指 binlog 文件写到一定大小后会切换到下一个，并不会覆盖以前的日志。
