mit6.s081 - xv6启动过程

核心问题：xv6实验中，执行 make qemu CPUS=1 后 xv6是如何启动运行的？

RISC-V的3种工作模式

在了解启动流程之前，首先需要明确RISC-V的3种工作模式

  1. M-mode (Machine Mode)
  2. S-mode (Supervisor Mode)
  3. U-mode (User Mode)

M-mode (Machine Mode)

M-mode是最底层的模式，也是每一个标准 RISC-V 处理器必须要实现的模式，它拥有最高权限，这意味着他将使用物理地址直接运行在硬件上。当cpu加电后，将处于M-mode。

机器模式具备拦截和处理异常的能力，并且可以访问所有其他模式下的控制状态寄存器CSR （Control Status Register）。

S-mode (Supervisor Mode)

Supervisor Mode（S-mode），如下图的RISC-V Privilege Levels中所示，是Machine Mode（M-mode）上两层的模式。这意味着操作系统调用OpenSBI的抽象来完成非常底端的操作。这样会使在开发板上开发一个操作系统变得容易一些。在此模式下，运行如操作系统内核的软件，并通过OpenSBI接口调用底层设备。

SBI是一种接口规范，提供了RISC-V标准的S-mode OS与M-mode的SEE（Supervisor Execution Environment）的接口，一般来说操作系统不会直接接触硬件，而是通过调用SBI的接口。OpenSBI正如其名，是一个开源的SBI具体实现，通过遵守开源协议，任何用户都可以修改和使用。

xv6启动过程

Step 1: 将xv6内核加载入内存

当RISC-V计算机开机时，它会初始化自己，并运行一个存储在只读存储器中的boot loader。

loader将xv6内核加载到物理地址0x80000000的内存中。之所以将内核放在0x80000000而不是0x0，是因为地址范围0x0:0x80000000包含I/O设备。

Step 2: 机器模式下运行，设置一个栈，便于跑C程序（汇编）

在机器模式下CPU从 _entry（kernel/entry.S:6）开始执行xv6。RISC-V在禁用分页硬件的情况下启动，此时虚拟地址直接映射到物理地址。

_entry 处的指令设置了一个栈，这样xv6就可以运行C代码。xv6在文件 start.c (kernel/start.c:11)中声明了初始栈的空间，即stack0。在 _entry 处的代码加载栈指针寄存器 sp，地址为 stack0+4096，也就是栈的顶部，因为RISC-V的栈是向下扩张的。现在内核就拥有了栈，_entry 调用 start (kernel/start.c:21)，并执行其C代码。

Step 3: 机器模式下运行，进行一些必要的配置，并切换到监督者模式（C语言）

函数 start 执行一些只有在机器模式下才允许的配置，然后使用指令 mret 切换到监督者模式。

  1. 设置特定寄存器，使得可以 mret 切换到监督者模式。
  2. 把 0 写入页表寄存器 satp 中，禁用虚拟地址转换。
  3. 把所有中断和异常委托给监督者模式
  4. 对时钟芯片进行编程以初始化定时器中断
  5. start 通过调用 mret 前往到监督者模式。

指令 mret 常用来从上一次的调用中返回(一般而言，上一次调用指的是从监督者模式转换到机器模式)，但是启动过程至今就是在机器模式，还没有进入监督者模式。

这里有一个很trick的地方：xv6在寄存器 mstatus 中把上一次的特权模式设置为监督者模式，并把 main 的地址写入寄存器 mepc 中，把返回地址设置为 main 函数的地址，这样使用指令 mret 就能切换到监督者模式了。

Step 4: 监督者模式下运行，操作系统初始化

  • consoleinit：设置console
  • kinit：设置页表分配器（page allocator）
  • kvminit：设置虚拟内存
  • kvminithart：打开页表
  • processinit：设置初始进程或者说设置好进程表单
  • trapinit/trapinithart：设置user/kernel mode转换代码
  • plicinit/plicinithart：设置中断控制器PLIC（Platform Level Interrupt Controller）
  • binit：分配buffer cache
  • iinit：初始化inode缓存
  • fileinit：初始化文件系统
  • virtio_disk_init：初始化磁盘

至此，操作系统已经基本运行起来了。

  • userinit：创建并运行第一个进程。

为什么要用init进程（pid=1）？
