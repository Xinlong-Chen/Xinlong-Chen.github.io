mit6.s081 - xv6物理内存分配器

核心问题：xv6是如何管理物理内存的？

RISV-V中的物理地址、虚拟地址

RISC-V 指令（用户和内核）操作的是虚拟地址。物理内存（RAM）是用物理地址来做索引的。

RISC-V的页表硬件（MMU）通过将每个虚拟地址映射到一个物理地址将这两种地址联系起来（地址映射）。

虚拟地址空间和物理地址空间的映射关系如下图：

从右边的物理地址空间格局可以看出其主要分为以 0x80000000 为界限的上下两个部分，分别是I/O设备地址和内存地址。

  • I/O：QEMU 在物理地址空间的 0x80000000 以下模拟 I/O 设备，如磁盘接口。QEMU 将设备接口作为内存映射（memory-mapped）的控制寄存器暴露给软件，内核可以通过读取/写入这些特殊的物理地址与设备进行交互。
  • 内存：QEMU 模拟的计算机包含 RAM（ 物理内存），从物理地址 0x80000000 开始， 至少到 0x86400000，xv6 称之为 PHYSTOP。内核对RAM和内存映射的设备寄存器使用直接映射（将这些资源映射到和它们物理地址相同的虚拟地址上）。

物理内存分配器（Physical memory allocation）

内核必须在运行时为页表、用户内存、内核堆栈和管道缓冲区分配和释放物理内存。

  • 物理内存管理范围：内核地址结束（kernel自 0x80000000 开始，内核可以管理kernel后的第一个地址）到 PHYSTOP 之间的物理内存。
  • 物理内存管理单位： 每次分配和释放整个4096 字节（4KB）的页面。
  • 数据结构：维护一个空闲页链表，记录哪些页是空闲的。
    • 分配物理内存：从空闲页链表中删除该节点；
    • 释放物理内存：将释放的页面添加到空闲页链表中。

物理内存管理的数据结构

内核通过维护一个空闲页链表，来记录哪些页是空闲的。

其定义如下：空闲链表由一个自旋锁保护，链表和锁被包裹在一个结构体中，以明确锁保护的是结构体中的字段。

物理内存初始化

在操作系统启动时，xv6在 main 函数中调用 kinit 来初始化分配器。

kinit 初始化空闲页链表，以保存内核地址结束（end）到 PHYSTOP 之间的每一页。

为什么是end 到 PHYSTOP 之间呢？

hard code，假设qumu模拟器就是这么模拟的，从而简化设计。

freerange 对每一页调用 kfree，使物理内存页面首地址加入到空闲链表中。

释放内存

kfree 就是一个链表添加操作。

新加入节点->next = 当前freelist头节点

freelist头节点 = 当前节点

需要注意的是，kfree 将被释放的内存中的每个字节设置为1，这可以加速有野指针的程序的崩溃。

分配内存

kalloc 就是一个链表删除一个节点的操作。

分配页面 = 当前freelist头节点

当前freelist头节点 = 当前freelist头节点->next
